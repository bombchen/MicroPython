# smoothed_led

`main.py` 是一个运行在 ESP8266 + MicroPython 上的 NeoPixel 灯带控制程序。它负责初始化 30 颗 LED，并根据设备能否连上已保存的 WiFi，在“控制模式”和“配置模式”之间自动切换。

## 主要功能

1. 初始化灯带

- 使用 `GPIO2` 驱动 NeoPixel 灯带。
- 默认灯珠数量为 `30`。
- 上电后先清空灯带，再初始化动画状态。

2. 记忆并尝试连接 WiFi

- 程序会从本地 `w.cfg` 读取 WiFi 的 `SSID` 和密码。
- 如果存在配置，就尝试以 STA 模式连接路由器。
- 最多等待约 `5` 秒；连接成功后进入控制模式，失败则进入配置模式。

3. 配置模式

- 当没有可用 WiFi 配置，或连接失败时，设备会启用 AP 模式。
- AP 名称固定为 `LED_Config`，无密码。
- 程序监听 UDP `8889` 端口，用于接收配网指令。
- 同时会定期扫描附近 WiFi，并每 5 秒通过 UDP 广播可用热点列表。

支持的配置指令：

- `config:SSID:PASSWORD`
  将 WiFi 信息写入 `w.cfg`，成功后回复 `OK!Rebooting...` 并重启。
- `status`
  返回 `CONFIG_MODE`。
- `list`
  返回最近一次扫描到的热点列表，格式为 `WIFIS:ssid1,ssid2,...`。

4. 控制模式

- 成功连上 WiFi 后，程序监听 UDP `8888` 端口。
- 主循环一边接收控制命令，一边持续刷新当前动画效果。

支持的控制指令：

- `mode:<effect>`
  直接切换到指定灯效。
- `mode:next`
  切换到下一个灯效。
- `mode:prev`
  切换到上一个灯效。
- `bright:<0-255>`
  调整全局亮度。
- `status`
  返回当前模式和亮度，格式为 `MODE:<mode>;BRIGHT:<value>`。
- `help`
  返回可用命令说明。

## 内置灯效

程序内置以下 8 种动画，并通过 `ANIM_FUNCS` 映射调度：

- `rainbow`：彩虹渐变循环。
- `breath`：红色全灯呼吸状态机。
- `fire`：随机火焰闪烁。
- `starry`：随机星点闪烁。
- `wave`：基于正弦波的颜色流动。
- `chase`：多色追逐尾迹。
- `sparkle`：随机彩色闪光。
- `snake`：带“食物”目标的贪吃蛇式移动效果。

## 程序运行流程

1. 创建 NeoPixel 对象并清屏。
2. 重置动画计数器和状态。
3. 尝试读取 `w.cfg` 并连接 WiFi。
4. 若连接成功，进入 UDP 控制模式。
5. 若连接失败，启动 AP 并进入 UDP 配置模式。

## 代码中的关键状态

- `mode`：当前灯效，默认是 `rainbow`。
- `brightness`：全局亮度，默认值是 `180`。
- `frame_count`：部分动画使用的帧计数器。
- `anim_state`：部分动画使用的运行时状态缓存。

整体上，这个程序的核心目标是：让 ESP8266 在首次未配网时通过 UDP 完成 WiFi 配置，配网成功后再通过 UDP 远程切换灯效和亮度，并持续驱动 NeoPixel 灯带播放动画。
